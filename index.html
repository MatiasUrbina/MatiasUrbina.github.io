Automated Electric Tests System

Project Type: Professional Project
Participation: Software Developer
Languages: LabVIEW and C++
Hardware: DC/AC Power Supplies, Power Analyzers, Digital Multimeters, Serial Communication Devices, Arduino and Relays
Objective: Provide to the diagnostic technicians a tool to improve their precision and accuracy in their diagnose to repair the product.

Description: The application is used to diagnose the product when any issue occurs during functional or final tests in the production lines. The application can run an automated test sequence to verify the product, if any test step fails during execution, the sequence will stop and show which step is failing, the test technician can use different virtual instruments included in the application to diagnose the product, repair it and return it to the production lines.

The hardware uses relays to switch between different instruments used during tests, to separate the communication when the product is being programmed, when there is a change in any electric input parameter and when the product is turned on.

It uses a centralized folder to call the information of any production line that will run the program and any product that will be tested. To enter in the application, it is necessary to log in, the is two types of users, a common user (test technician or operator) who can only use final test sequence and virtual instruments, and an administrator who can use all the options including functional test.

The application creates a log file that saves all the information from any sequence that has been runned, if the sequence fails, the first failed step and the values received are saved, and a report is created monthly, including a resume from all the tests and products tested.

Hotel Checkout

Project Type: College Project
Participation: Software Developer
Languages: LabVIEW
Hardware: myRIO, servomotor, and RFID reader
Objective: Create an application that allows to hotel clients to do an automated checkout using an RFID card.

Description: The application is used to provide hotel users a tool to do a quick checkout. When a client checks in the hotel, it is given an RFID (Radio Frequency Identification) card with all the information that is used by the hotel, the user can use this card to enter in the room that has been assigned and other areas where the user has access.

The application has a main screen (Home) with two button, one allows to change the language (Spanish/English) and the other button starts the application, when the user pushes this button, the application goes to the "Insert card" screen, where it is required to insert the card in the machine, after the card is inserted, the machine reads the RFID card and shows the "Welcome" screen where it is showed the total cost of the service, in this screen there is three buttons:

  • Checkout: The machine keeps the card and the user checkout from the hotel. The screen goes to "Checkout"
  • View detailed information: Goes to the "Detailed Information" screen, that shows all the information from the user.
  • Home: The user decides to stay in the hotel and the machine returns the RFID card. The screen goes to "Home"
If the user selects the View detailed information button, the screen "Detailed Information" shows how many days the user has been in the hotel, the price for every day, every service that has been used, the final price and more data. The "checkout" screen shows a farewell message and then goes to "Home" screen.

The machine is programmed in LabVIEW using a NI myRIO as a controller, when the RFID card enters in the machine, a servomotor keeps the card, if the user decides to checkout, the servomotor rotates 90° and the card falls in a box inside the machine, if the user decides to stay in the hotel, the servomotor rotates 45° and the card falls through a tunnel that returns it to the user.

Angular Robotic Arm

Project Type: College Project
Participation: Software Developer and Mechanical Designer
Languages and programs: C++, MIT App Inventor and SolidWorks
Hardware: Arduino, Bluetooth module HC-06 and servomotors
Objective: Create an Angular robotic arm controlled with a Bluetooth mobile application.

Description: The robotic arm has 3 Degrees of freedom, it can move its base, its main arm and forearm 180°, also it has a gripper that can open and close using a servomotor. The arm can be controlled changing the angles of the different servomotors, the robot is positioned in a 3D plane, it has X, Y and Z axis, and can be used entering coordinates.

When the robot is turned on, it comes to a "home position" and waits to the user to connect the phone. To use the robot the user must install the application, when the application starts it is necessary to press the "Buscar dispositivo Bluetooth" button (if the user has connected the robotic arm before, the application connects to the arm automatically), when the button is pressed, a window with all available devices will open and the user must select the HC-06 module.

To move the robotic arm using the angles of each servomotor, it must be used the "Cinemática Directa" controls:

  • q1: Is the angle of the servomotor located at the base of the robotic arm.
  • q2: Is the angle of the servomotor located at the main arm of the robotic arm.
  • q3: Is the angle of the servomotor located at the forearm of the robotic arm.
To move the robotic arm using X, Y and Z coordinates, it must be used the "Cinemática Inversa" controls:
  • Px: It is the position in the X axis where the arm can move.
  • Py: It is the position in the Y axis where the arm can move.
  • Pz: It is the position in the Z axis where the arm can move.
After the positions or angles has been introduced, the user must press the "run" button, and the robot will move first its base, then its main arm and finally the forearm. The gripper can be used with the "gripper" button, it only has two positions, open and close

The application was build using MIT App Inventor, all the formulas for the direct cinematic and inverse cinematic were calculated by the application, after the angles were calculated, they were sent to the Arduino that receives the information using the HC-06 module and the Serial Buffer, and finally moves the servomotor to the calculated angles. The gripper also has a servomotor that only has two positions, one angle for open and the other for close.

The robotic arm was designed using Solidworks and it was build using 3D printing in PLA plastic.
